# Карта Скиллов взаимодействия (Guiding Principles)
## Текущая детерминированная версия (Фиксация: Границы Циклов и 4 Уровня)

### 1. Архитектурное правило версионирования
* Каждая новая итерация или модификация кода конвертера обязана содержать инкрементальный номер версии в конце имени файла (например, `py2drn5.py`, `py2drn6.py`, `py2drn7.py`).
* Категорически запрещается повторно использовать имя или перезаписывать индекс уже зафиксированной, обсужденной или протестированной версии. При любом изменении внутренней логики индекс строго повышается на единицу.

### 2. Запрет на превентивную генерацию кода (Протокол согласования)
* При переходе на новый шаг, обнаружении топологических нестыковок или получении замечаний **категорически запрещено** превентивно выводить программный код новой версии.
* **Строгий алгоритм действий:**
  1. Сформулировать предлагаемое решение исключительно текстовым, инженерным языком.
  2. Подробно объяснить планируемые геометрические, топологические или семантические изменения.
  3. Дождаться явного текстового подтверждения или прямой команды (например, *«Давайте попробуем»*) от пользователя.
  4. Только после получения одобрения предоставить листинг следующей версии конвертера.

### 3. Приоритет изоляции на тест-кейсах
* При отладке базовых физических принципов трансляции (построение линий, выравнивание осей, парсинг абстрактных синтаксических деревьев) работа ведется исключительно в изолированном контексте файлов атомарного примера — **`rmsnorm.*`** (`rmsnorm.py`, `rmsnorm.drn`).
* Переход к масштабным, многосвязным или ветвящимся структурам (например, `microgpt.py`) допускается только после полной фиксации идеального результата на атомарном примере.

### 4. Топологический детерминизм (Закон Многолистных Римановых Пространств)
* Каждый изолированный кусок алгоритма (функция) разворачивается на отдельном двухмерном листе (`diagram_id`), образуя многолистную Риманову поверхность проекта в реляционной БД.
* **Лист 1 (Главный, индекс функции 0):**
  * Центральная ось шампура: строго $X = 500$, интервал высот $Y \in [420, 750]$.
  * Выносные параметры: мост от $X = 530$, икона параметров в $X = 690$ ($Y = 420$).
  * Флаг состояния элементов: `selected = 1`. Камера холста: `origin = '0 250'`.
* **Лист 2 и последующие (Дополнительные, индекс функции >= 1):**
  * Смещение в левый верхний угол: центральная ось шампура строго $X = 170$, интервал высот $Y \in [20, 350]$.
  * Выносные параметры: мост от $X = 200$ длиной $w = 140$, икона параметров в $X = 360$ ($Y = 20$).
  * Флаг состояния элементов: `selected = 0`. Камера холста: `origin = '-41 -82'`.
* **Глобальное состояние проекта (`state`):** фиксирует фокус на последней созданной странице (`current_dia = 2`), переключая интерфейс на актуальный лист алгоритма.

### 5. Многомерный Тензор Декомпозиции Алгоритма (Проекция на Железо и Смысл)
При анализе и трансляции каждой функции учитывается её распределение по 9 фундаментальным уровням, где первые четыре уровня жестко проецируются на графические примитивы языка ДРАКОН:
1. **The object** $\rightarrow$ реализуется через **икону Action (Процесс)**.
2. **Upgrade to the states. Finite State Machine (FSM)** $\rightarrow$ реализуется через **икону Choice (Выбор / Переключатель)**.
3. **To implement the switching functions / equalization / branching** $\rightarrow$ реализуется через **икону Question (Вопрос)**.
4. **Function parser (parsing) / compilation** $\rightarrow$ реализуется через парную конструкцию **икон Begin for Loop / End for Loop**.
5. **By field of memory and the physical devices** (Поля памяти, адресация и физические периферийные устройства).
6. **By time intervals** (Временные интервалы, квантование и тайминги выполнения).
7. **Search function / insert data into the repository** (Функции поиска, индексации и вставки данных в репозиторий/хранилище).
8. **Along the route objectives / Target way** (Маршрутизация целей и траектория целевого пути алгоритма).
9. **By the user and planned processes / The sensitivity analysis** (Пользовательские и планируемые процессы, анализ чувствительности системы).
